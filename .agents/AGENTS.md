# Workspace Rules & Context: Polaris (HookDrop)

Welcome to **Polaris** (historically named HookDrop). This project is a lightweight webhook capture and streaming utility written in Go, bundled with a production-style Kubernetes infrastructure, GitOps sync, security, and observability stack.

---

## ⚠️ Isolation Constraint

- **Strict Separation:** This workspace is **completely separate and isolated from the AegisMesh IAM project**. 
- Never mix namespaces, dependencies, configurations, or coding paradigms from AegisMesh IAM into this repository.
- Avoid referencing AegisMesh IAM files, endpoints, or features in any PRs, commits, documentation, or code logic created in this workspace.

---

## 🛠️ Tech Stack & Architecture

- **Go Backend:** Written in standard Go, using `net/http` for routing and `github.com/rs/zerolog` for structured logging.
- **In-Memory Store:** Webhooks are captured in memory. Stored events are capped at a maximum of `50` events per bucket (configured in `store/memory.go`). A server/pod restart clears all stored webhooks.
- **SSE Streaming:** Real-time push capability using Server-Sent Events (SSE) via `/h/<bucket-id>/stream`. Includes periodic keepalive messages (`: keepalive\n\n`) every 15 seconds to keep proxies/load balancers active.
- **Observability & Tracing:**
  - Integrates with OpenTelemetry. When `OTEL_EXPORTER_OTLP_ENDPOINT` is configured, request spans are exported over OTLP gRPC.
  - **Dual Trace IDs:**
    1. **Transport-level Trace ID:** Managed by request logging middleware (`requestLogger` in `main.go`). It uses `X-Trace-Id` from HTTP headers if present, or generates a UUID.
    2. **Per-event Trace ID:** Generated by the webhook post handler (`handlePost` in `handler/webhook.go`) and embedded inside the payload.
- **Infrastructure & GitOps:**
  - Local setup uses **Docker Desktop Kubernetes** (context verification via `scripts/setup-cluster.sh` or `scripts/setup-cluster.ps1`).
  - Packaging uses **Helm** (`helm/hookdrop/`).
  - Deployments are managed via **ArgoCD** GitOps (`k8s/argocd/`).
  - Security policies (e.g., Cosign image signature verification) are enforced at the cluster boundary via **Kyverno** (`k8s/kyverno/`).
  - Monitoring stack includes Prometheus, Loki, Tempo, Grafana, and an OTel Collector (`scripts/setup-observability.sh`).

## 📝 Coding Guidelines

- **Maintain Safety:** The store uses `sync.RWMutex` to ensure thread-safe operations on buckets and subscription channels. Any changes to `store/` must maintain proper mutex locking/unlocking.
- **Preserve Documentation & Comments:** Keep all existing comments, docstrings, and telemetry configurations intact when modifying Go code.
- **Telemetry Compliance:** Ensure any new handlers wrap requests with the `requestLogger` middleware and propagate the context correctly.
